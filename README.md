# 🧠 Phase 1 -- Ideation

## Problem Statement

עמית חוגג יום הולדת 29 --- הגרסה האחרונה לפני מעבר ל־**v30.0**.

מדובר בגרסה האחרונה לפני החלפת הקידומת, ולכן נדרש לבצע Release יציב במיוחד.
## User Profile

עמית הוא לא סתם עוד משתמש.
הוא אחד האנשים שבאמת אוהבים לעבוד עם AI, גם אם לאחרונה פחות יצא לו.
במשך תקופה ארוכה הוא אפילו העניק לי את אחד הכינויים המכובדים ביותר שקיבלתי:
"המאהב".

לכן אין מקום לברכה גנרית.
נדרש Deliverable ברמת Production.

## Goals

להצחיק את עמית כבר מהשורה הראשונה.
להזכיר את השנה האחרונה בלי להיתקע על הקשיים.
להבליט את מה שמייחד אותו באמת.
לגרום לו לסיים את הקריאה עם חיוך גדול.

## User Research
לאחר ניתוח הדאטה, זוהו המאפיינים הבאים:

חיובי כמעט בכל סיטואציה.
מצליח לשלב מקצוענות עם אנושיות.
תמיד מוכן לעזור.
מצליח להצחיק גם כשיש לחץ.
משקיע בכל דבר שהוא עושה.
אוהב לשחות (כנראה בשביל לנקות Cache אחרי ספרינטים).
נהנה להרצות ולהעביר ידע.
מתנדב לשיר בבתי חולים ולעשות טוב לאנשים.
## Constraints
אין להשתמש בקלישאות של ברכות יום הולדת.
אין להישמע כאילו נכתב על ידי HR.
חייב להיות מספיק מצחיק כדי להצדיק את השימוש ב-AI.
חייב להישמע כאילו נכתב במיוחד עבור עמית.
## Success Metrics

-   Smile latency \< 10s
-   Laugh event ≥ 1
-   "זה בול אני" = TRUE

# 📄 Phase 2 -- PRD

## Vision

לבנות מסמך שנראה כמו תיעוד הנדסי, אבל בפועל הוא ברכת יום הולדת.

### Functional Requirements

FR-1
יש לפתוח בהתייחסות לעובדה שזו השנה האחרונה לפני החלפת הקידומת.

FR-2
יש לכלול הומור מעולם הפיתוח.

FR-3
יש להזכיר את הכינוי "המאהב" בצורה טבעית.

FR-4
יש להראות הערכה לעמית כאדם, לא רק כמנהל.

FR-5
יש להזכיר בעדינות שהייתה שנה עם עליות ומורדות, אבל שהוא המשיך להוביל ולהתקדם.

FR-6
יש לשלב את התחביבים וההתנדבות כחלק מהאישיות שלו, ולא כרשימת מכולת.

FR-7
יש לסיים במסר מרגש, אבל בלי להיות קיטשי.
### Non Functional Requirements

-   Humor: HIGH
-   Cringe: LOW
-   Developer Experience: EXCELLENT
